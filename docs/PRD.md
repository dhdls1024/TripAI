> **문서 버전**: v1.0
> **작성일**: 2026-04-10

> **트랙**: AI/빅데이터 트랙
> **수행 기간**: 2026. 4. 10. ~ 2026. 6. 16. (약 9주)

## 1. 개요 (Overview)

### 1.1 배경 및 필요성

- 한국관광공사 빅데이터에 따르면, 전국 관광지 방문자 수는 2020년 약 2,450만 명에서 2025년 약 3,100만 명으로 지속 상승 중이며, 신용카드 기반 관광지출액 또한 2021년 약 145조 원에서 2024년 약 184조 원으로 증가
- MZ세대(20~45세)의 여행 수요가 빠르게 회복되고 있으며, 핵심 트렌드는 다음과 같음:
  - 소규모·개별 여행 선호 (1~2인 여행 비중 약 75%)
  - 가격 민감성과 할인 프로모션 반응도 증가
  - 수도권을 벗어난 비일상 여행지 선호
  - 여행 경험의 적극적 공유 문화
- 기존 여행 플래너는 일정 생성·예산 관리·정산·기록 등 각 단계가 **분절**되어 사용자가 다수의 앱을 병행해야 하는 불편함이 존재
- 생성형 AI를 활용한 **전 사이클 통합 지원 서비스** 개발이 필요

### 1.2 프로젝트 비전

> 사용자 적재 데이터를 기반으로 AI API를 활용하여 **국내 여행의 전체 사이클(계획 → 준비 → 여행 중 → 여행 후)**을 AI가 지원하는 **통합 여행 플래너 웹 플랫폼** 개발

### 1.3 참고자료

| 출처 | 제목 |
|------|------|
| 한국관광공사 데이터랩 | 빅데이터(이동통신) 지역별 방문자수 |
| 한국관광공사 데이터랩 | 빅데이터(신용카드) 지역별 관광지출액 |
| 한국관광공사 관광컨설팅팀 | 「2024 관광트렌드 전망 및 분석」, 2023.12 |
| 한국관광공사 관광데이터전략팀 | 「MZ세대 숙박여행 트렌드 분석」, 2024.01 |

## 2. 목표 (Goals)

### 2.1 최종 목표

사용자 적재 데이터를 기반으로 AI API를 활용하여 국내 여행의 전체 사이클을 AI가 지원하는 통합 여행 플래너 웹 플랫폼 개발

### 2.2 세부 목표

| 구분 | 범위 | 내용 |
|------|------|------|
| 세부 목표 1 | **MVP** | 카카오톡 로그인 API 연동 및 사용자 데이터 관리 시스템 구축 |
| 세부 목표 2 | **MVP** | AI API 연동을 통한 AI 국내 여행 일정 자동 생성 기능 구현 |
| 세부 목표 3 | **MVP** | 친구 초대, 공동 플래닝 및 그룹 역할 분담 기능 구현 |
| 세부 목표 4 | 향후 고도화 | 여행 중 실시간 일정 재조정, 예산 관리 및 AI 공정 정산 기능 개발 |
| 세부 목표 5 | 향후 고도화 | AI 여행 회고록 자동 생성 및 게시판 공유/타인 여행지 가져오기 기능 구현 |

### 2.3 성공 지표 (KPI)

- 카카오톡 로그인을 통한 회원가입 및 인증 정상 동작
- AI 여행 일정 자동 생성 요청 대비 정상 응답률 95% 이상
- WebSocket 기반 실시간 공동 플래닝 보드 동시 접속 안정성 확보
- MVP 기능 전체 통합 테스트 통과
- AWS 배포 및 서비스 가용성 확인

## 3. 사용자 (Users & Personas)

### 3.1 타깃 사용자

- **주 타깃**: MZ세대(20~45세) 국내 여행자
- **세부 유형**:
  - **플렉스+형**: 높은 소비 의향, 프리미엄 경험 추구
  - **가치에 누리는형**: 가성비 중시, 할인 프로모션 반응도 높음
  - **호시탐탐형**: 비일상 여행지 탐색, 새로운 경험 선호

### 3.2 사용자 시나리오

#### 시나리오 1: 개인 여행 계획
> 대학생 A가 "겨울 방학 기간, 3~4일 일정으로 항공편을 이용해 부산 신라호텔 숙박을 포함한 20대 대학생의 힐링 여행"을 자연어로 입력하면, AI가 맞춤형 동선 및 예산안을 자동 도출

#### 시나리오 2: 그룹 여행 공동 플래닝
> 친구 4명이 제주도 여행을 계획할 때, 방장이 플래닝 보드를 생성하고 친구들을 초대. 각자 희망사항을 입력하면 AI가 합의안을 생성하고 투표를 통해 최종 일정 확정

#### 시나리오 3: 여행 후 회고록 공유
> 여행 완료 후 사진·메모·방문장소를 기반으로 AI가 스토리텔링 여행기를 자동 작성. 게시판에 업로드하여 다른 사용자와 공유

## 4. 기능 요구사항 (Functional Requirements)

### 4.1 1단계: 기반 시스템 구축

#### FR-1.1 카카오톡 OAuth2.0 소셜 로그인
- 카카오톡 OAuth2.0 기반 소셜 로그인 페이지 개발
- 별도 비밀번호 저장 없이 보안 로그인 구현
- 회원 탈퇴 시 개인정보 즉시 삭제 처리

#### FR-1.2 사용자 데이터 모델 및 DB 설계
- 사용자 프로필, 국내 여행 이력, 선호도 데이터 모델 및 ERD 설계
- 개인정보보호법 준수를 위한 최소 수집 원칙 적용 및 데이터 보유 기간 명시

#### FR-1.3 기본 UI 구현
- 플랜 작성, 공유, 모니터링 기본 화면 구현
- 토스·카카오페이 등 국내 대표 핀테크 서비스의 UI/UX 트렌드 반영:
  - **카드형 모듈 UI**: 일정·예산·정산 정보를 카드 단위로 분리하여 대시보드 구성
  - **원스텝 인터랙션**: 한 화면에서 한 가지 핵심 행동만 유도하는 단계별 질문 방식
  - **엄지 친화적 하단 네비게이션**: 한 손으로 쉽게 조작 가능한 버튼 배치
  - **다크모드·라이트모드**: 사용자 환경에 맞는 테마 자동 적용
  - **마이크로 애니메이션**: 버튼 클릭·AI 응답 대기 시 부드러운 피드백 제공

### 4.2 2단계: AI 핵심 기능 개발

#### FR-2.1 AI 여행 일정 자동 생성
- 사용자 입력(국내 여행지, 날짜, 예산, 취향) 기반 프롬프트 엔지니어링 설계
- AI API 호출을 통한 시간대별 일정, 맛집/숙소 추천, 동선 최적화 기능

#### FR-2.2 AI 여행지 추천 엔진
- 자연어 입력만으로 조건에 맞는 국내 여행지 추천 및 이유 설명

#### FR-2.3 AI 예산 플래너
- 항목별(교통·숙소·식비·관광) 예산 배분안 및 절약 시나리오 생성

#### FR-2.4 AI 테마별 여행 코스 설계
- 맛집 투어, 자연 힐링, 역사 탐방 등 테마 코스 생성

#### FR-2.5 동행자 성향 매칭 일정
- 동행자별 선호도 반영 절충 일정 및 분리 동선 제안

### 4.3 3단계: 확장 기능 개발

> **MVP 범위 내 구현**: 1~2단계 + 3단계의 '[소셜] AI 공동 플래닝 보드'
> **향후 고도화 과제**: 여행 중 실시간 재조정, 공정 정산, 회고록 자동 생성

#### FR-3.1 [여행 준비] AI 짐 싸기 체크리스트
- 여행지 날씨·활동·기간 분석 → 맞춤 패킹리스트 생성
- 날씨 기반 구체적 안내 포함

#### FR-3.2 [여행 준비] AI 국내 예약·준비물 가이드
- 관광지 사전 예약, 숙소·교통편 예약, 축제 티켓팅 등 체크리스트 정리
- 마감일 기준 역산 타임라인 자동 생성

#### FR-3.3 [여행 준비] AI 지역 정보 카드
- 교통카드 사용 여부, 주차 정보, 해수욕장 안전구역·산불 위험 기간 등 정리
- PDF 저장 기능으로 현장 활용 가능

#### FR-3.4 [여행 중] 실시간 일정 재조정 _(향후 고도화)_
- 기상청 단기예보 API(날씨) 및 카카오 모빌리티 API(교통) 연동
- 기상 악화·변수 발생 시 AI가 남은 일정을 실시간 재배치

#### FR-3.5 [여행 중] AI 맛집·카페 즉석 추천 _(향후 고도화)_
- 현재 위치 기반 리뷰 데이터 + 위치 정보 결합 추천
- 카카오 API 연동 지도·길찾기로 경로 최적화

#### FR-3.6 [여행 중] AI 여행 비용 실시간 트래커 및 공정 정산 _(향후 고도화)_
- 지출 입력 시 남은 예산 대비 소비 속도 분석 및 조언 생성
- 정산 방식:
  - 1/N 균등 분할
  - 차량 이동 정산 (연료비 자동 계산)
  - 차량 기여도 정산 (운전 시간/거리 기반 분배)
  - 참여도 기반 정산 (불참자 비용 자동 제외)
- AI가 가장 공정한 분배 방식 자동 제안
- 정산 결과 카카오톡 공유 또는 개인별 자동 전송
- 여행 지출 카테고리별 자동 분류 및 파이차트 시각화
- 예산 대비 소진율 게이지 바 실시간 표시, 초과 임박 시 알림

#### FR-3.7 [여행 중] AI 긴급상황 도우미 _(향후 고도화)_
- 분실·부상·차량 고장 등 긴급 상황 시 대응 방법과 연락처 즉시 안내

#### FR-3.8 [여행 후] AI 여행 회고록 자동 생성 및 게시판 공유 _(향후 고도화)_
- 사진·메모·방문장소 기반 스토리텔링 여행기 작성
- 블로그·SNS·개인용 톤 선택 가능
- 게시판 업로드로 다른 사용자와 공유
- 타인의 여행지/일정을 내 플랜으로 가져오기 가능

#### FR-3.9 [여행 후] AI 여행 통계 리포트 _(향후 고도화)_
- 총 지출·이동 거리·방문 장소 자동 집계
- 카테고리별 소비 인사이트 인포그래픽 스타일 리포트 생성

#### FR-3.10 [소셜] AI 공동 플래닝 보드 _(MVP 범위)_
- **"여행 깃허브"** 개념: 여행 일정을 팀원들과 실시간으로 작성·수정·공유
- WebSocket 기반 실시간 양방향 데이터 동기화
- 친구 초대, 희망사항 수집, AI 합의안 생성 및 투표 기능
- 그룹 역할 분담: 방장·운전자·회계·일정 담당
- 일정 충돌·비용 불균형을 AI가 선제 감지해 분쟁 전 조정안 제시

### 4.4 4단계: 통합 테스트 및 배포

#### FR-4.1 통합 테스트
- 전체 기능 통합 테스트 및 사용자 시나리오 테스트 수행
- 게시판 부하 테스트 및 동시 접속 안정성 검증

#### FR-4.2 성능 최적화
- API 응답 속도 최적화 및 캐싱 전략 적용

#### FR-4.3 배포
- AWS EC2, S3, Nginx 기반 배포 및 운영 환경 구축

## 5. 비기능 요구사항 (Non-Functional Requirements)

### 5.1 보안
- 카카오 OAuth2.0 기반 인증 (별도 비밀번호 저장 없음)
- 사용자 여행 이력·지출 데이터 **AES-256 암호화** 저장
- **HTTPS** 통신 적용
- 개인정보보호법 준수: 최소 수집 원칙 적용 및 데이터 보유 기간 명시
- 회원 탈퇴 시 개인정보 즉시 삭제 처리

### 5.2 성능
- AI API 응답 시간 최적화 (캐싱 전략 적용)
- WebSocket 기반 실시간 동기화 지연 최소화
- 게시판 동시 접속 안정성 확보

### 5.3 UX/UI
- 토스·카카오페이 UI 트렌드를 참고한 직관적·심플한 디자인
- 연령·디지털 숙련도에 관계없이 사용 가능한 최고 수준의 편의성
- 다크모드·라이트모드 지원
- 모바일 최적화 (엄지 친화적 하단 네비게이션)

### 5.4 확장성
- 사용자 여행 이력·선호도·소비 패턴 데이터를 누적·분석하여 추천 정확도 지속 고도화
- AI 생성 데이터(일정, 추천 결과, 회고록)를 별도 테이블에 저장하여 추후 학습 데이터로 활용

## 7. 데이터 모델 개요

### 7.1 데이터 유형

| 데이터 유형 | 설명 | 저장 방식 |
|------------|------|----------|
| 사용자 입력 데이터 | 국내 여행 선호도, 예산, 여행 이력 | DB 적재 |
| AI 생성 데이터 | 일정, 추천 결과, 회고록 | 별도 테이블 (학습 데이터 활용) |
| 게시판 데이터 | 회고록 게시글, 좋아요, 여행지 가져오기 이력 | DB 관리 |
| 여행 중 실시간 데이터 | 지출 내역, 방문 장소 체크 | 실시간 동기화 |

### 7.2 주요 엔티티 (ERD 설계 대상)

- **User**: 사용자 프로필, 카카오 인증 정보
- **TravelPreference**: 여행 선호도, 취향 데이터
- **TravelHistory**: 과거 여행 이력
- **TripPlan**: 여행 계획 (AI 생성 일정 포함)
- **Schedule**: 시간대별 세부 일정
- **Budget**: 예산 항목별 배분
- **PlanningBoard**: 공동 플래닝 보드
- **BoardMember**: 보드 참여자 및 역할
- **Vote**: 투표 항목
- **Expense**: 지출 내역
- **Settlement**: 정산 내역
- **Memoir**: 여행 회고록
- **Post**: 게시판 게시글

## 11. 차별화 포인트

### 11.1 기존 서비스 vs 본 프로젝트 비교

| 구분 | 기존 여행 플래너 | 본 프로젝트 (AI 통합) |
|------|----------------|---------------------|
| 일정 생성 | 사용자가 직접 검색/수립 | AI가 취향/예산 기반 자동 생성 |
| 예산 관리 | 단순 기록/정산 | AI가 항목별 배분 및 절약 전략 제안 |
| 비용 정산 | 1/N 균등 분할 | AI가 참여도/기여도 기반 공정 정산 자동 제안 |
| 일정 변경 | 사용자가 직접 수정 | AI가 상황 분석 후 실시간 재배치 |
| 여행 준비 | 사용자가 직접 확인 | AI가 예약/준비물/지역 정보 자동 정리 |
| 긴급 대응 | 사용자가 직접 검색 | AI가 상황별 대응 매뉴얼 즉시 생성 |
| 여행 후 기록 | 수동 작성 | AI가 데이터 기반 회고록 자동 생성 |
| 경험 공유 | 링크 공유 수준 | 게시판 공유 + 타인 여행지 가져오기 |
| 데이터 활용 | 일회성 정보 제공 | 누적 데이터로 추천 정확도 지속 향상 |
| 소셜 기능 | 단순 초대 | "여행 깃허브" 실시간 공동 편집 + 역할 분담 |

### 11.2 3대 핵심 차별화

1. **여행 전 사이클의 AI 완결 구조**: 계획→준비→여행 중→여행 후의 전 과정을 단일 플랫폼에서 AI가 연속적으로 지원
2. **AI 공정 정산 시스템**: 차량 기여도, 참여도, 연료비 자동 계산 등 상황별 공정 분배 방식을 AI가 자동 제안
3. **"여행 깃허브" 실시간 공동 플래닝**: WebSocket 기반 양방향 동기화, 그룹 역할 분담, AI의 충돌/불균형 선제 감지

## 14. 활용 방안 및 기대 효과

### 14.1 활용 방안

- **개인 여행자 대상**: AI 기반 맞춤형 여행 일정 생성 서비스, 데이터 축적에 따른 개인화 추천
- **그룹 여행 활용**: AI 공동 플래닝 기능으로 의사결정 효율화, 카카오톡 연동 초대/공유
- **커뮤니티 활용**: 회고록 게시판을 통한 여행 경험 공유, 타인 여행지 가져오기, 로컬 여행 정보 확산

### 14.2 기대 효과

#### 기술적 측면
- AI API 프롬프트 엔지니어링 및 컨텍스트 관리 기술 역량 확보
- 사용자 데이터 기반 AI 추천 시스템 설계 및 구현 경험 축적
- 실시간 데이터 처리 및 API 연동 기반 웹 서비스 개발 능력 향상
- 게시판 CRUD, 데이터 공유, 소셜 기능 등 풀스택 개발 역량 강화

#### 사회경제적 측면
- 여행 계획~정산까지 하나의 앱에서 완결되는 통합 국내 여행 관리 경험 제공
- "여행 깃허브" 개념의 새로운 여행 협업 문화 제시
- AI 공정 정산으로 그룹 여행의 금전 갈등 구조적 해소
- 카카오 API 연동으로 별도 앱 설치 없이 친숙한 환경에서 즉시 사용 가능

#### 산업적 측면
- 생성형 AI 기반 국내 여행 서비스의 새로운 비즈니스 모델 제시
- 여행업계의 AI 전환을 위한 참고 사례
- 회고록 게시판 + AI 플래너 결합 모델의 새로운 방향 제시
